# 核心优化测试指南

## 测试环境准备

1. 启动后端服务器
```bash
cd backend
npm start
```

2. 启动前端开发服务器
```bash
cd frontend
npm run dev
```

3. 打开浏览器访问 http://localhost:5173

---

## 测试项目清单

### ✅ Phase 1: 基础设施测试

#### 1.1 知识图谱系统
**测试目标**: 验证实体和关系追踪

**测试步骤**:
1. 创建一个新游戏（任意类型）
2. 开始游戏后，进行以下操作：
   - 与 NPC 对话
   - 访问不同地点
   - 获取物品
3. 打开浏览器控制台，检查是否有知识图谱相关日志

**预期结果**:
- 角色、地点、物品被正确添加到知识图谱
- 关系被正确建立（如：角色A认识角色B）

#### 1.2 事件时间线
**测试目标**: 验证事件记录和查询

**测试步骤**:
1. 在游戏中进行多个回合（至少10个）
2. 执行不同类型的动作：对话、探索、战斗
3. 检查控制台是否记录了事件

**预期结果**:
- 每个回合的事件被记录
- 事件包含参与者、地点、重要性等信息

#### 1.3 因果链
**测试目标**: 验证因果关系追踪

**测试步骤**:
1. 执行一个有明显因果关系的动作（如：攻击敌人 → 敌人反击）
2. 检查控制台日志

**预期结果**:
- 因果关系被正确记录
- 可以查询"为什么会这样"

---

### ✅ Phase 2: 核心引擎测试

#### 2.1 流式输出
**测试目标**: 验证文本逐字显示

**测试步骤**:
1. 开始游戏
2. 输入任意动作（如："我向前走"）
3. 观察旁白文本的显示方式

**预期结果**:
- 文本逐字显示（类似 ChatGPT）
- 不是一次性全部显示
- 显示流畅，无卡顿

**如何验证**:
- 打开浏览器开发者工具 → Network 标签
- 查看 `/api/games/{gameId}/action` 请求
- 应该看到 `Content-Type: text/event-stream`

#### 2.2 一致性验证
**测试目标**: 验证生成内容不与设定冲突

**测试步骤**:
1. 创建一个游戏，记住角色名字和设定
2. 游戏中尝试使用不存在的物品
3. 尝试去不存在的地点
4. 检查控制台是否有一致性警告

**预期结果**:
- 控制台显示一致性验证警告
- 系统拒绝不合理的操作或自动修正

#### 2.3 智能上下文压缩
**测试目标**: 验证长对话不会丢失关键信息

**测试步骤**:
1. 进行长时间游戏（至少20个回合）
2. 在第20回合提到第1回合的事件
3. 观察 AI 是否记得

**预期结果**:
- AI 能记住重要事件（即使超过6回合）
- 不重要的细节可能被压缩

#### 2.4 选项预生成
**测试目标**: 验证选项响应速度

**测试步骤**:
1. 开始游戏，等待第一个选项出现
2. 等待5秒（让预生成完成）
3. 点击一个选项
4. 观察响应速度

**预期结果**:
- 第一次点击选项：正常速度（5-10秒）
- 第二次点击选项：几乎瞬间响应（使用预生成结果）

**验证方法**:
- 控制台应该显示 "预生成完成: {选项文本}"
- 点击时显示 "使用预生成结果"

---

### ✅ Phase 3: 高级功能测试

#### 3.1 并行图像生成
**测试目标**: 验证图像生成不阻塞文本

**前提**: 需要配置图像生成服务（ComfyUI 或其他）

**测试步骤**:
1. 启用自动图像生成
2. 输入动作
3. 观察文本和图像的显示顺序

**预期结果**:
- 文本立即开始流式显示
- 图像在后台生成，完成后显示
- 文本显示不等待图像

#### 3.2 记忆持久化
**测试目标**: 验证记忆在重启后保留

**测试步骤**:
1. 玩游戏10个回合
2. 保存游戏
3. 刷新页面
4. 加载游戏
5. 提到之前的事件

**预期结果**:
- AI 记得之前的事件
- 知识图谱、时间线、因果链都被恢复

## 性能测试

### 响应时间对比

**测试方法**:
1. 关闭所有优化（修改代码临时禁用）
2. 测量10个回合的平均响应时间
3. 启用所有优化
4. 再次测量10个回合的平均响应时间

**预期改进**:
- 流式输出：感知延迟减少 80%（虽然总时间相同，但用户立即看到反馈）
- 选项预生成：第二次点击响应时间减少 90%
- 智能压缩：长对话性能提升 30%

## 质量测试

### 一致性测试

**测试场景 1: 角色记忆**
1. 第1回合：告诉 NPC "我叫张三"
2. 第10回合：问 NPC "我叫什么名字？"
3. **预期**: NPC 回答 "张三"

**测试场景 2: 物品追踪**
1. 第1回合：获得"魔法剑"
2. 第5回合：使用"魔法剑"攻击
3. 第10回合：尝试再次使用"魔法剑"
4. **预期**: 系统知道你有这把剑

**测试场景 3: 地点连贯性**
1. 第1回合：在"森林"
2. 第2回合：走到"村庄"
3. 第3回合：尝试直接去"城堡"（没有路径）
4. **预期**: 系统提示无法直接到达，或自动生成路径

```javascript
this.debug = true;
```

## 已知问题和限制

1. **流式输出**: Anthropic API 暂不支持流式，会回退到普通模式
2. **预生成**: 只预生成当前选项，不预生成更深层的分支
3. **图像生成**: 需要额外配置 ComfyUI 或其他服务
4. **记忆容量**: 知识图谱在超过10000个节点后可能变慢
